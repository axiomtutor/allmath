Primitive objects in Python

Enter

In[1]: 1+1

and the console should show

Out[1]: 2

In[2]:

The console is now waiting for the next instructions. The commands you issue are numbered, but depending on what you do, your numbers may disagree with mine. I'll use ^^In[n]:`` and ^^Out[n]`` to stay agnostic about line numbering. Now try running the following.

In[n]: 'hello world'

Out[n]: 'hello world'

In this example I've used the text object 'hello world'. Python distinguishes text objects, called "strings", from number objects. In the first example of 1+1 the number 1 is called an "integer". It is used to distinguish numbers like these from numbers like 1.5 which we call "floats".

In[n]: 1.5*4

Out[n]: 6.0

There is just one more so-called primitive object type that we will need for a while, and that's a boolean, meant to represent true and false.

In[n]: not True

Out[n]: False

In[n+1]: not False

Out[n+1]: True

The use of boolneans will, hopefully, become clear soon and their nature will be explored in greater detail in the topics on Logic and Computer Science. For now just appreciate that + and * are operations on numbers, and in a similar way not is an operation on booleans. Another operation is ^^and`` and this is an infix operation so that you have to write, for instance

In[n]: True and False

Out[n]: False

There are more operations, and we'll introduce them as they become useful. Strings also have operations, here's an illustration of concatenation.

In[n]: 'Hello ' + 'world'

Out[n]: 'Hello world'

Python intelligently detects what kind of object you're using, in this case a string, and does the right thing for that kind of object. Notice that sometimes you can get slightly unexpected results like

In[n]: 2*'Hello'

Out[n]: 'HelloHello'

A priori you might have guessed that multiplying a number and a string would produce an error. Sometimes operations like that does produce an error.

In[n]: 'Hello' + 1

Traceback (most recent call last):



File "<ipython-input-1-d8d11a7979bf>", line 1, in <module>

'Hello'+1



TypeError: Can't convert 'int' object to str implicitly

The error messages can sometimes be intimidating. However, often the error type--in this case, TypeError--is helpful and simple. This is the error type you get when you do something to a thing and the thing doesn't like it. In this case the number doesn't like getting a string added to it--Python doesn't know what you want when you do that, so it gives you an error.

Variables

Everything we've done so far creates an object, like a number or text, and after that it's gone--we can't access what we created without entering it again. We want the ability to store the results of our work and re-use it, and for this we have variables. Every variable is an information storage container, with a name. The name of the is always in un-quoted text, like so

In[n]: sto = 0

In[n+1]: sto + 1

Out[n+1]: 1

Here the name of the variable is sto and it has been made to store the number 0. Unexciting but it's a start. This is typical in Python: Whenever we create a variable we have to simultaneously create it and store some initial information in it. In this example, sto was "initialized" to 0. If we wanted to illustrate multiplication as repeated addition, for instance showing 3*4 as adding 3 with itself 4 times, we could do the following. This code first stores 3 then adds 3 to itself four times. At the final input line, it asks the consol to print with the simple command ^^sto``.

In[n]: sto = 3

In[n+1]: sto = sto + 3

In[n+2]: sto = sto + 3

In[n+3]: sto = sto + 3

In[n+4]: sto

Out[n+4]: 12

This code might be confusing, because mathematically sto = sto + 3 is a contradiction: There is no number which is equal to: itself plus 3. But recall that, somewhat unfortunately for us, in Python the = symbol does not express equality, it expresses information storage.

Let's dissect what line In[n+1]: sto = sto + 3 does (I really do mean the ^^n+1`` part--that's where I'm picking up the explanation here). Reading the right-hand side first, ^^sto+3`` retrieves the value in sto, which is 3, and sums it with 3 to get 6. Finally, the ^^sto =`` tells Python to store this value, 6, back into sto, in the process, erasing the 3 that was there before. Again, make sure you think of variables, here, as storage containers and not mathematical variables. The following does the same thing, using some "syntactic sugar" in the form of +=.

In[n]: sto = 3

In[n+1]: sto += 3

In[n+2]: sto += 3

In[n+3]: sto += 3

In[n+4]: sto

Out[n+4]: 12

To be clear, the line ^^In[n+1]: sto += 3`` is perfectly equivalent to ^^In[n+1]: sto = sto + 3``. At this point you should feel free to do some exercises if you feel like it. For instance, illustrate repeated multiplication; or try to understand what True or False does and what it means. Try to play around with operations like 2**3 and see if you can figure out what it does and what it means.

Lists

So far every object we've dealt with has just been of a primitive type, more or less, but sometimes we want to build a complex object out of primitive parts. An important example is a list, which just takes a bunch of objects and stores them in one place.

In[n]: lister = [10, 5, 9]

The "one place" that I mean, is ^^lister``. That is to say, ^^lister`` is the one storage location, holding all three of these data points. One use of lists is to, for instance, compute the sum of the list.

In[n+1]: sum(lister)

Out[n+1]: 24

which if we wanted to we could further save and use to find the mean of the list.

In[n+2]: s = sum(lister)

In[n+3]: s/len(lister)

Out[n+3]: 8.0

You could probably guess that the ^^len()`` function gives the length of the list. Now it's great to be able to box a bunch of information together, but it's common to want to pull elements back out--so-called "list indexing". For this we can use the bracket notation after the list.

In[n+4]: lister[0]

Out[n+4]: 10

In[n+5]: lister[1]

Out[n+5]: 5

It is likely obvious by now that Python uses "zero-indexing" meaning that we start counting from 0. Because of zero-indexing, the following will always throw and error no matter what list we store in lister

In[n]: lister[len(lister)]

but the following will always produce the last element of the list

In[n]: lister[len(lister)-1]

You would not be wrong to think this is a kind of obnoxious, verbose way of getting the last element, which is kind of a "natural" and frequent thing you want to access. Isn't there a way to save key-strokes? Yep! Python really is beautiful and friendly:

In[n+1]: lister[-1]

Out[n+1]: 9

Strings as lists

Strings are in fact lists of characters __Hide: characters are individual glyphs__. We can index them just like lists:

In[n]: exampleString = 'Here is an example of some random text'

In[n+1]: exampleString[1]

Out[n+1]: 'e'

We saw how we could use the len() function to find the length of the list and use that to find the last element. However, both strings and lists allow you to do this with negative indices.

In[n+2]: exampleString[-1]

Out[n+2]: 't'

In[n+3]: [10,9,8,7][-2]

Out[n+3]: 8
