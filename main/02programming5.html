Modules

NumPy

We will frequently use the SymPy module. A module is a collection of code, usually absolutely massive, that someone else has written. You're able to install it and then use that code without ever having to look at it. One fairly easy module to learn is the NumPy module so we'll study that before studying NumPy.

NumPy, short for numerical Python, is a module written to help you use advanced mathematical features. Copy and run the following script.

1| import numpy as np

2| print( np.sin(1) )

3| print( np.sin( np.pi/2 ) )

4| x = np.array([1, 2, 3])

5| print( x.var() )

Line 5 computes the variance of the list--a statistic that we often compute on data. But the point here is just about how to import and use a module. Line 1 imported the module, ^^import numpy`` and then gave it a shorter name with ^^as np``. I could have chosen most any other name, but almost everyone chooses ^^np`` for NumPy, although I've never been able to figure out why ...

Anyway, on line 2 what you see is that NumPy (or np) has the sine function from Trigonometry implemented. To access this function we write ^^np.sin()``. I've chosen to compute the sine of 1 radian and print it on this line. NumPy's trigonometric functions assume angle measures in radians. On line 3 I again use NumPy's sine function, but this time I give it an argument of TODO/2. However, Python doesn't have a TODO value out-of-the-box, so I have to get this too from NumPy. In order to get it, I use ^^np.pi``. You can print this on its own and see the decimal expansion of TODO.

On line 4 I exhibit the use of a NumPy array. A NumPy array is a lot like a list (in fact, to create one, you give ^^np.array()`` a list for an argument). However, a list is a simpler object. A NumPy array supports many extra functions. On line 5 I demonstrate the use of one of these, the ^^.var()`` method.

However, NumPy is numerical in its essence. You might think, well yeah, so is Math so hip-hip-hooray for that. But Math isn't fundamentally numerical, it's fundamentally conceptual, and we use symbols to represent abstractions. When we talk about `sqrt{2}` we do not want to talk about the decimal 1.4 although this is kinda close. What we really want is ... well ... we want `sqrt{2}`, there's just no better way to say it. We want this symbolic expression. So-called numerical methods are good for applications--engineering, finance, and so on. But they're not good for Math. Therefore we will rarely use NumPy and instead rely much more on SymPy: Symbolic Python.

SymPy As advertised, the following script will not print the decimal expansion of `\sqrt{2}` or TODO.

1| import sympy as sp

2| print( sp.sqrt(2) )

3| print( sp.pi )

This might seem useless, and a little bit it is. However, notice that the following script prints exactly the right result, rather than one correct to many decimals:

1| import sympy as sp

2| print( sp.sqrt(2)**2 )

These sorts of examples, though easy to produce, are actually very far from the most interesting demonstrations of what SymPy can do. Many of the things we want to do with SymPy requires the use of variables--not CompSci variables, but mathematical variables. To produce these you need the ^^sp.symbols()`` function, which I demonstrate below.

1| import sympy as sp

2| x = sp.symbols('x')

3| print(x)

4| print(x**2)

5| sp.plot(sp.sin(x))

Notice that the third line would have produced an error if the second line had not first been run. Notice that line 3 prints the symbolic representation of the function `x^{\wedge 2}`, "x-squared". Line 5 is responsible for a pop-up window showing you the graph of the sine function. Even these don't really show you the value of SymPy.
