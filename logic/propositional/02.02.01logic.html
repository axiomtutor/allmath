Exercise: You've probably wondered what the ^ symbol does in Python if not compute an exponent. It turns out that it computes the exclusive-or! Try running True ^ True and True ^ False. You can even try it on numbers like 0 and 1. Generally in Computer Science we associate the number 0 with false and the number 1 with true. ^ will behave exactly as you expect it to on such numbers. However, on any other numbers, it will give results that are likely confusing and surprising. I won't go into the explanation, but it suffices to say that it involves a conversion of the decimal number into binary, and then computes the XOR on 0's and 1's, before converting back to decimal at the end.



Answer: True ^ True returns False, and True ^ False returns True.

Question: It should be pretty obvious that disjunction is associative and commutative, although knock yourself out with a truth-table to demonstrate it if you like. Instead, this exercise will focus on connections between the three symbols, ¬, ∧, and ∨.

(i) What will our laws of precedence be, now that we have a new symbol? We decided it for negation and conjunction, and decided that negation will have narrow scope.

(ii) Does any distribution law hold for ¬(𝕡∨𝕢)? For ¬(𝕡∧𝕢)?

(iii) Does any distribution law hold for 𝕡∨(𝕢∧𝕣)? For 𝕡∧(𝕢∨𝕣)?

Answer: (i) We will again choose the negation to have the narrowest scope, but that doesn't resolve what we decide for the competition between disjunction and conjunction. We still need, for instance, a decision about how to read 𝕡∨𝕢∧𝕣. Typically we just won't write something like this because even with a precedence rule, it gets hard to read without parentheses. However, in some cases we will rely on precedence, and assume that 𝕡∨𝕢∧𝕣 means 𝕡∨(𝕢∧𝕣). That is to say, conjunction takes precedence over disjunction (and negation takes the highest precedence of them all).

(ii) Yes, and if you're puzzled about how to see it, think in terms of a natural language expression. If you say, in some kind of malformed English, "Not ( Russia is large or Costa Rica is large )" first let's clean this up and better express it as "Neither Russia nor Costa Rica is large". Now clearly this proposition is equivalent to the conjunction "Russia is not large and Costa Rica is not large". Therefore we might expect that ¬(𝕡∨𝕢) is equivalent to ¬𝕡∧¬𝕢. If you draw the truth-table for these you will see that they are equivalent. Likewise ¬(𝕡∧𝕢) is equivalent to ¬𝕡∨¬𝕢. This equivalence is called the distribution law.

(iii) Yes, and this one is not so easy to motivate by a natural language example, but it still makes some intuitive sense. 𝕡∨(𝕢∧𝕣) should be true (or false) in the same situations as (𝕡∨𝕢)∧(𝕡∨𝕣). Both of these are made immediately true whenever 𝕡 is. If 𝕡 is false, then in both cases the their truth requires both 𝕢 and 𝕣 to be true. Again, a truth-table will definitely prove the equivalence. This is called de Morgan's law.

Question: If two sentences have the same truth-table, are they therefore equivalent?

Answer: At first glance the answer seems like a trivial "yes" because that's practically what the definition of equivalent is. However, is the truth-table for 𝕡 (i.e. just the table with T, F down the column) the same as the truth-table for 𝕢 (also with just T, F down its column but with 𝕢 in the first row instead)? What about 𝕡∨(𝕢∧¬𝕢) and 𝕡 which we've seen are equivalent, but the former has four rows and the latter just two? Do these have the same truth-table?

The answer is no, but clearly if we collect together all of the atoms in either sentence and build the truth-tables from those, that brings greater uniformity. Moreover, if we only focus on the truth-values under the main connective, this gives us a good tool for comparing sentences. Therefore if 𝕡 is any proposition, atomic or possibly complex, and 𝕢 is another, we define their atom set to be the set of all atoms that occur in either of the two. We write the atom set as A(𝕡, 𝕢).

For example, in the propositions 𝕡 = ¬(𝕡1 ∧ 𝕡2 ∧ ¬( ¬ 𝕡1 ∨ 𝕡3 )) and 𝕢 = 𝕡2 ∨ 𝕡1 ∨ 𝕡3 then we have A(𝕡, 𝕢) = {𝕡1, 𝕡2, 𝕡3}. Notice that I will use double-struck letters for atoms and underlined, double-struck letters for possibly complex sentences.

Then we define the truth-vector of a proposition 𝕡 under an atom set A. This is list of truth-values that appear under the main operator in its truth-table, where the truth-table is drawn with all atoms in A. So for instance the truth-vector of ¬𝕡 is just [F, T] if the atom set is just {𝕡}. However, the truth-vector is {F,F,T,T} if the atom set is {𝕡,𝕢} for instance.

Exercise: Using the concept of the truth-vector, give a necessary and sufficient condition for two sentences being equivalent.

Answer: It is necessary and sufficient that whenever propositions 𝕡 and 𝕢 are equivalent, they have the same truth-vector under atom set A(𝕡, 𝕢).

Exercise: What is the relationship between a structure, a proposition, and the proposition's truth-vector?

Answer: The structure determines one of the values in the truth-vector, by determining that row of the truth-table.

Exercise: Suppose you are given any truth-vector of length 2^n. What must be the size of the atom set?

Answer: An atom set of n elements generates a truth-table of length 2^n and therefore this is the length of the truth-vector. Thus the atom set must have n elements.

Exercise: Suppose you are given an arbitrary truth-vector of length 2^n. Can you always find a sentence which produces this truth-vector under an atom set of size n?

Answer: Yes, it's kind of a tricky process when you first learn it, but basically what you do is form the sentence which "asserts" all of the structures where the truth-vector is true. For example take the truth-vector [T,T,F,T,F,F,F,F] under atom set {𝕡, 𝕢, 𝕣}. To build the sentence that we want, we can "assert" the first structure because we want a sentence that is true there. That is to say, we assert 𝕡∧𝕢∧𝕣 because this corresponds to the first structure where 𝕡 and 𝕢 and 𝕣 are all true. We also assert the second structure, by which I mean we form the proposition 𝕡∧𝕢∧¬𝕣 as the second structure is true for 𝕡 and 𝕢 but false for 𝕣. We skip the third structure because the truth-value is false in the third coordinate of the vector. We assert the fourth structure, 𝕡∧¬𝕢∧¬𝕣, and we're finished with the individual assertions.

Finally we chain all of the assertions into a disjunction. We get the sentence (𝕡∧𝕢∧𝕣) ∨ (𝕡∧𝕢∧¬𝕣) ∨ (𝕡∧¬𝕢∧¬𝕣). If you draw the truth-table of this sentence you'll see that it has T only in the first, second, and third structures. This is called disjunctive normal form. There is also a conjunctive normal form but we won't discuss it.

Question: Next symbol?

Answer: This one attempts (poorly) to capture the notion of "if ... then ..." For example, when trying to represent the sentence "If the circle has a radius of 2 then it has a circumference of 4π" we will write 𝕡→𝕢 where of course 𝕡 represents "the circle has radius 2" and 𝕢 represents "the circle has circumference 4π. We call 𝕡 the antecedent of the if-then sentence and 𝕢 the consequent. We will shortly see how the framework we have built is inadequate to capture this meaning in most natural language contexts. We have to decide how we will evaluate this operation in terms of truth-value. The least controversial decision is to suppose that, if 𝕡ℳ = T and 𝕢ℳ = T then (𝕡→𝕢)ℳ = T. After all, if the circle really does have radius 2 and really dies have circumference 4π then you really do regard the if-then sentence as a true one. The next consideration should be fairly incontroversial, if 𝕡ℳ = T and 𝕢ℳ = F then we should say that (𝕡→𝕢)ℳ = F. After all, in saying, "if 𝕡 then 𝕢" is kind of like a promise that, should you find 𝕡 is true then you will also find that 𝕢 is. If you instead find that 𝕢 is false, then you may regard the promise as broken and the if-then sentence therefore false.

After these, there are only bad options. What if 𝕡ℳ = F? Since the if-then is like a promise conditional on 𝕡 being true, then when you find that 𝕡 is not true it is as if you made no promise at all. How should we evaluate the truth of the if-then in this case, though? We could, and perhaps should, say that it has no value in this case. However, if we were to adopt that convention then further analyses would become very inconvenient and awkward. __Hide__ The most useful convention is to suppose that such a sentence is true. Therefore in both cases where 𝕡ℳ = F and 𝕢ℳ = T, and also where 𝕡ℳ = F and 𝕢ℳ = F, we will say that (𝕡→𝕢)ℳ = T. That is to say: 𝕡→𝕢 is true when the antecedent and consequent are both true, and also true when the antecedent is false.

It is a bit funny, and it mis-matches natural English language in a lot of examples (according to this analysis "If pigs fly then 1+1=3" is a true sentence because the antecedent is false). However, to get a more realistic analysis requires a LOT, and it doesn't aid our study of Mathematics, so we skip it.
