Chapter 02 - Extending number systems

Section 04 - p-adic numbers

Terms: p-adic digit, p-adic integer, strictly negative, p-adic rationals, p-adic numbers

Symbols: Zp, Qp,

Question: It probably will occur to the reader that, given all of these choices that we've made, first to represent numbers as digits, and then close systems of numbers under operations, and then close them under digit sequences ...


Was there another good choice that we could have made instead?


Answer: That's a hard question to answer in its full generality given how very open-ended it is. Still it's a good question that turns out to have at least one important positive answer: p-adic numbers. These have only been studied for the last century-and-change in response to certain specific mathematical needs, but their use has been growing in recent years. The subject is cool but skipable given how new and exotic it is.


Definition: If p is a prime number, all of the numbers 0, 1, ..., p-1 we call p-adic digits. Take for example 31-adic digits. These are 0, 1, 2, ..., and so on up to 30. However 31 and 32 are not 31-adic digits.


Intuitively, initially we will think of this as just the same thing as a base-system as we've studied it, where the base has to be a prime number. For instance we've already done this quite a lot with base-2, but here we will consider also base-3, base-5, etcetera. Thus if we use 7-adic numbers, the number 132 would represent 1*7^2 + 3*7^1 + 2*7^0 or if written in base-10 this would be 49+21+2 = 72. We will be switching the base representation of numbers so often that I won't attempt to a unique formatting to the text of each p-adic representation.


We call a p-adic integer representation any number of the form a0a1a2...an if each ai is a p-adic digit. There is no operation going on between the digits (yet), this is just some listing of p-adic digits. So for instance 12,10,0,14 is a 17-adic integer representation, where I've thrown in commas to help see the separation of digits. The number that we associate with this representation, the p-adic integer, regards these digits as coefficients of powers of p. Thus 12,10,0,14 regarded as a 17-adic integer is equal to `12\ast 17^3 + 10\ast 17^2 + 0\ast 17 + 14`. In decimal representation we would call this number 61860.


Of course this is just base-p representation so far, just specifying that p must be a prime. Well we've been doing that all along with p = 2 although under this description we might be motivated to try other primes. That is to say, rather than constructing negative numbers as those with a _ symbol in front, we let the digits of the number carry on to infinity. In doing so however, we ensure that the negative of any 2-adic integer is an infinite sequence of 1s except for some collection of 0s close to the early digits. So for instance _2 in 2-adic numbers is ...11110 because if you sum this with 10 (which is the 2-adic integer for the number 2) then you get ...00000 or just 0.


There we illustrated with 2-adic integers, but for any prime p the p-adic negative of a natural number will have an infinite collection of digits p-1 followed by a collection of finitely many digits that may not be p-1. So for another example the 3-adic negative of 1 is ...22222 and the 3-adic negative of 2 is ...22221 and the 3-adic negative of 3 is ...22220 and so on. We call these numbers "strictly" negative.


One funny upshot of permitting digits to carry on infinitely to the left, though, is that some of them are not strictly negative even with these infinite non-zero digits. For instance in 2-adic numbers ...10101010 is a number that's neither positive nor negative, strictly speaking, although it clearly cancels with ...110110 so that these two are relatively negative, that is to say, negative to each other in the way that _2 is negative to 2 since they sum to 0.


To define multiplication we effectively just extend the table algorithm that we described before. For any of these funny neither-negative-nor-positive numbers, you can only compute their digit expansion in finite time and not fully discover them, but for some you can still give a full description. A trivial example is that in 2-adic numbers the number ...0101 * 10 = ...1010 and since you can figure out the pattern in the digits then you don't need to compute until infinite to fully describe the digit sequence.


We call this set of numbers Zp. It will likely be no surprise, given how integer-y this thing is, that division is not defined on it just like integer division is not defined in the usual way for certain numbers. In particular, in Z2 the number 1/10 (i.e. the number which when multiplied by 10 returns 1) does not exist--again I reiterate that I'm writing in 2-adic so that 1/10 in decimal is 1/2. The shock comes from the fact that this (and things like it, such as 1/100 which in decimal would be 1/4) are the only numbers that are not necessarily divisible. What this is telling us is that i) one-third in 2-adic integers is ...111 and ii) what we constructed as rational numbers using regular base systems, are neither strictly positive nor strictly negative in this system. I.e. in 2-adic integers, the ratio 1/11 or "one-third" is neither positive nor negative. Of course this is a strong departure from how we ordered the rational numbers in section 2 of this chapter, which was based off of a good ordering of the integers, and which made up our number line. You'll have to have patience before you can discover why any of this matters.


Exercise: Find ...3334 + ...3334 in 7-adic integers. What can you conclude about this number, in terms of it being some kind of ratio?

Answer: ...3334 + ...3334 = ...0001 = 1. Thus in 7-adic numbers ...3334 = 1/2.

Question: We won't prove it now but it turns out that the p-adic integers ending in 0 are the ones which do no have reciprocals in the integer system. What happens if, like in the base system that we explored before, we close the system under the operation of division?

Answer: We get a genuinely new number system, of course, the p-adic numbers, denoted Qp. Amusingly (to me) the rationals were numbers that had finitely many non-zero numbers to the left but could have non-zero digits infinitely to the right. For instance 3.333... is a rational number but not ...3333. In our case, it will turn out that we can associate with every p-adic number a digit expansion with an infinity of digits to the left but finitely many to the right!
