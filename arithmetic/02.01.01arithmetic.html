Chapter 02 - Extending number systems

Section 01 - Integers

Abstract: Integers are the natural numbers together with negatives. They can be constructed mathematically as the closure under subtraction or intuitively as modeling steps from a privileged position. After their construction we have to define operations in a i) consistent and ii) good way.

Terms: element, natural number, closure, set builder notation, integer, positive, negative, absolute value, equivalence relation

Symbols: ℕ, {, }, :, _, ℤ, ℤ+, ℤ-, _n, |.|, ≡

Question: What is 0-1?

Answer: So far, this means nothing. The only numbers that exist for us are 0, 1, 2, ... It will be a helpful description to call these numbers natural numbers and label the set of all such numbers as ℕ. We often express sets with curly braces so that the set of numbers 0, 1, 2 is {0,1,2}. Let's call this list A so that A = {0,1,2}. All this does is collect the elements together into a group, in a kind of analogy to how lists work in CompSci, except that lists are ordered. A list like list1 = [0,0,1,1] has four elements and a coordinate system so that I can specify the "zeroth" and "oneth" elements of the list. list1[0] is, with the Python notation, the number 0. list2[1] is also 0. This behavior is unlike sets, which are unordered. There is no such thing as the zeroth or first element of A. It doesn't have a coordinate system. Therefore the set {0,0,1,2} is just the same as the set {0,1,2} because all we can say is the set contains 0--we cannot distinguish the "first" zero from the "second". The only distinguishing features of sets are membership. We can tell {1,2,3} from the set {5,4,3} because the former has 1 and the latter doesn't. We cannot tell {1,2,3} from {3,2,1} because all sets are the same regardless of ordering. We call the contents of a set elements or members.

Now back to the question, as I said, for us right now, all that exists in the world of numbers is ℕ and in this number set, there is no number such that it, plus 1, is 0. However, We could create a number like this, call it _1. This _1 will denote a newly created number which when added to 1 produces 0. Let's investigate the properties that _1 must have in order to be consistent with the rest of our numbers.

To begin with, what should _1 + 2 be? Well, since _1 + 2 = _1 + 1 + 1 and we know the first part of that, _1 + 1, is equal to 0, then this must simplify to 0 + 1 = 1. Hence we have just established that _1 + 2 = 1 and we could likewise show that _1 + n = n - 1 for all natural numbers n. But here's a twist:

Question: What is _ + _?

Answer: Again this cannot exist as any of the numbers we've encountered so far. It has the property that, when added to 2, sums to 0. We could again create a new number, which we could reasonably name as _2 to suggest that it's _ twice.

Rather than creating a new number over and over, we could instead say that we want the "closure" of all numbers under subtraction. That is to say, if x and y are any natural numbers, then we call x+y a (possibly new) number. Thus we get 5-1 is a number--which we already knew--and we get 2-5 is a number, and so on for all choices of natural number. Let us define these terms more rigorously now.

Definition: If @ is any operation (addition, subtraction, multiplication, etc) then we say that the closure under @ is the set of all z such that z is equal to some x @ y. In the notation of set theory we would say that the closure under @ is {z : z = x @ y for some natural numbers x,y}. The : symbol is read as "such that". This is called set builder notation and generally has the format {<expression> : <condition>}. To give another example we could define the set of even natural numbers as E = {2x : x is any natural number}. Notice that 0 is in E because 0 = 2*0. Also 2 is in E because 2 = 2*1. In this way, E will contain all and only the even natural numbers.

We define the integers as ℤ = {z : z = x - y where x, y are some natural numbers}. Thus 5-1 and 2-5 are in ℤ. To give an example of something not in ℤ, 2/5 is not, because there are no natural numbers x, y such that 2/5 = x - y. We define _ as 0-1 (or, if you prefer, _1 is equal to 0-1) and _2 as 0-2, and in general _n is 0-n which by definition has the property that, when summed with n produces 0. Every integer that's not a natural number we call a negative, and the set of negatives we denote ℤ- which implies ℤ- = {_1, _2, _3, ...}. We call ℤ+ the positives, which are non-zero natural numbers, i.e. ℤ+ = {1, 2, 3, ...}.

The choice of ℤ as the symbol for the integers comes from the fact that they are Zahlen, the German word for "numbers".

Question: Well, so be it, we have integers now. What do we do with them?

Answer: We need to define operations of +, -, *, and so on, until we are able to sensibly compute these on all "numbers". Notice by the way that suddenly the word "number" has become ambiguous: Natural number? Integer? If we're incautious you may need to figure out the intended number set by context, but we should be careful to try to specify which numbers we mean in the future.

So let's take for instance _5 and figure out what is _5 + x when integer x is greater than 5 and when it's less than 5 (when it's equal to 5, we know the sum is 0 by definition). When x is greater than 5, like for instance 6, the case is easy. We can just argue like so, _5 + 6 = _5 + 5 + 1 = 0 + 1 = 1. That is to say, when x > 5 we can just split off 5 from x and cancel.

If x < 5 like say 4 we could compute the following. _5 + 4 = _5 + (5 - 1) = 0 - 1 = _1. Of course this requires an item of faith, that associativity allows us to drop the parentheses at the penultimate step. In a way we already accepted this article of faith in the previous calculation, writing the sum of three things without parentheses at all--assuming that this made sense and addition remains associative even with negative numbers.

In a sense it's not an article of faith, it's much worse. It's that there is no fact of the matter one way or another--but in defining what we mean by negative numbers, we define them to satisfy all of the previous properties that the natural numbers have: Commutativity, associativity, distributivity, and the like. And in fact, this state of affairs is just how we should have expected it to be: We are doing everything so abstractly, using the closure under subtraction to generate our numbers.

What we're doing here is taking a more physical or applied perspective than the abstract perspective begun above that used closure. Here we start with a description of a system we want, and then think of the Mathematical objects or rules that have these features. Either perspective is fine, and it's probably good to have each in mind to help reinforce the other.


So if this is our physical system, a sequence of dots extending to infinity to the right and left, with 0 as a privileged "starting" position, this could be like a one-dimensional line counting steps east and west from home. The plus operation takes a certain position and walks more to the right, the minus operation takes a position and walks to the left. This could also be like a bank account where money is "positive" and debt is "negative".


In any case we will want addition to be commutative and associative no matter what the exact physical interpretation is. 3 - 4 should mean three steps right and four left; _4 + 3 means four left, three right; they should both arrive at _1. Similarly 3 + _4 should do the same thing as 3 - 4. They mean subtly different things but still calculate to the same number. It may be helpful to think of positive numbers as always "facing" or being oriented toward the right and negative numbers as facing or being oriented toward the left. Then 3 + _4 could be thought of as being at 3, and turning around (facing left) and moving forward by 4 (arriving at _1). On the other hand 3 - 4 could be thought of as being at 3, keep facing to the right, but walk backwards by 4--still arriving at _1. The main difference to keep track of is that _ is not an operation while - is. _1 is a number or a symbol to indicate left-ward orientation.


In any case, the point here being that the physical interpretation suggests to us that the addition operation should have all the familiar properties that we know and love--making this requirement seem less arbitrary than it might in the abstract context. Thus the above demonstration that _5 + 4 = _1 is valid in both views.


One issue we might have is how to add two negatives, like _1 + _1. The physical perspective easily justifies that this is _2. Abstractly we could argue that _1 + _1 has the property that when summed with 2 it results in 0, and so does _2. If we require that only one number can have this property then we can argue _1 + _1 = _2. If these arguments feel ad hoc don't worry: We will review these constructions again in Set Theory and Abstract Algebra in a more systematic way.

Question: What should multiplication mean in this context?

Answer: Of course repeated addition is sensible when the negative appears on the left. _3 * 4 = _3 + _3 + _3 + _3 = _12

When a negative number occurs in the right factor though, we seem to need to make some sense of adding "negative-many" times. What should 2 * _2 be? Abstractly we would only require multiplication to be commutative just like it is in the case of natural numbers, so that we could justify saying this is the same as _2*2 = _4. Physically, things are more difficult, but thinking in terms of velocities could help. Suppose we have an object moving at 60 kilometers per hour for 3 hours, then of course it travels 60*3 = 180 kilometers. We could regard the time 0 hours to be right now, and 3 hours to represent 3 hours in the future, so that in a sense this calculation represents a prediction. We can then make sense of negative time as just being time in the past, before now, and the 60*_3 = _180 kilometers represents a retrodiction: three hours ago the car was 180 kilometers to the left. This perspective in terms of velocity also coheres with the notion of negative velocities and negative time. It even tells us what to do to multiply two negatives: You just retrodict motion toward the left and therefore the location must be to the right. So _60*_3 = 180.

The abstract justification is much trickier (notice how some things are handled trivially by the abstract perspective while physically they're tough, and vice versa for some other things). We could argue in the following way, assuming that multiplication has all the nice properties, commutativity, associativity, distributivity, and the idea that 0 times any number must be 0.

$$0 = 1 + \_1 \Rightarrow \\ 0*\_1 = (1 + \_1)*\_1 = 0 = 1*\_1 + \_1*\_1 = \_1 + \_1*\_1 $$

From this we infer that _1*_1 has the property that when summed with _1 it results in 0. This is unique to the number 1, so we have _1*_1 = 1.

Question: What about division?

Answer: The answer of course comes from what we already say about multiplication. For instance, _4/2 is that number which when multiplied by 2 produces _4. Of course this number must be _2. Other cases are equally obvious.

Question: What is a complete algorithm for adding, subtracting, multiplying, and dividing integers?

Answer: It will be helpful to have the concept of the absolute value to state this.

Definition: The absolute value of x, written |x| is the "positive part" or "size" of x. Formally we may define this piecewise: If x is a natural number, |x| = x, otherwise |x| = _x. Notice that the absolute value is always a natural number. For example |2| = 2 but |_2| = 2.

Now for an algorithm to find x+y,

  • If both are natural numbers then of course add as in the last chapter.

  • If one is negative, we can assume "without loss of generality" that it is x. What this "without loss of generality" means is that, if it were actually y that's negative, we could just swap the numbers around and the same proof would work. This "without loss of generality" claim comes up so often, we abbreviate it w.l.o.g. Anyway, again w.l.o.g. we assume x is negative.

    • If |x| > |y| then find |x| - |y| using the usual subtraction process on natural numbers, and the value of x+y will be _(|x|-|y|).

    • Otherwise the value is |y|-|x|.

  • If both are negative the value is _(|x|+|y|).

For subtraction, x-y = x+_y so that the addition algorithm suffices.

For multiplication x*y is

  • The usual if they are natural numbers

  • _|x|*|y| if one is negative

  • |x|*|y| if both are negative

For division, x/y is just like for multiplication: In all cases, divide the absolute values. If they're the same sign, the result is positive; otherwise the result is negative.

Question: How do we exponentiate?

Answer: Of course again the easy part is to have a negative on the left, in which case we perform repeated multiplication. _2^2 = _2*_2 = 4. The hard part is when a negative is on the right: 2^_2

One way to motivate this is to look at the progression as we go from 2^2 = 4 to 2^1 = 2 to 2^0 = 1. As the exponent decreases the value is half as large. Since there is no number half as large as 1, though, we have to leave this undefined ... for now.

Inverses of exponentiation are no more interesting than this, so we won't pursue those now.
