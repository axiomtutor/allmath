Arithmetic

Chapter 02: Extending number sets

Section 03: Real numbers

Abstract: The real numbers plug all gaps in the number line.

Terms: number line, dense, real

Symbols: ℝ

Question: If sqrt(2) isn't rational then what do we say about it?

Answer: The obvious answer is to close our number system under this operation as we have with other operations. However, that doesn't quite get us as much as we might think it does. We know that eventually we're going to have to encounter and explain π, the constant used for measuring circles, but this number is not going to be the square-root of a rational number or anything like it. Yet it has a famous decimal expansion, 3.14somethingsomething I don't really know or care about the digits of π. Don't get me wrong, I deeply love π, I just don't care about its digit representation.

Now is probably a good moment to introduce the notion of the rational number line. This is like the labeled dot sequence that we used for the integers, now fitting in a lot of points in-between.

__Example__

This number line clearly has a lot more--and infinity more--points than the integer number line, which itself had an infinity more points than the natural number line. (For the mathematicians out there, don't give me crap about what I've said here--there is a sense in which this is true using set-minus, and I'll introduce bijections later.) But not only does the set of rational numbers have more than the integers, but in-between any two integers there is an infinity of rationals.

__Example__

In fact, in-between any two rationals there is an infinity of rationals. This is a geometric property that we call density, defined as: For any two numbers, r < s there exists a number t such that r < t < s. Notice that the digit approximation of sqrt(2), which we said was 1.414213... and this gets closer and closer to a place that's ... "on" the rational number line? Well, saying that it gets close to a place on the rational number line seems wrong because we already said that sqrt(2) is not rational. Maybe we should say that it gets close to a place in-between the rational number line? That also seems weird because there doesn't seem to be space in-between stuff in a dense geometry like this. But apparently there is. This is also strange, which is what I love about Mathematics, that even in a dense space like this, it still has gaps! Clearly to fill the gaps we will need to include all digit sequences. This and not the closure under square-roots will get us the real numbers, denoted by ℝ. That is to say ℝ is the set of all digit sequences, so that 102.3456789013423423456... ∊ ℝ and any other digit sequence is in it too. If we want to state it just a little more formally we could say

$$\mathbb{R}=\{a_ma_{m-1}...a_{0}.b_1b_2... : \text{ for all } i \in \mathbb{N} \text{ we have } a_i, b_i\in \mathbb{N} \text{ and } 0\leq a_i,b_i\leq 9\}$$

But that's just a lot less readable than "all digit sequences". It basically says that the real number is every sequence with m characters, followed by a dot, followed by another sequence of characters, such that the indices of the characters are natural numbers and represents a digit.

Anyway, this has the consequence that, now, sqrt(2) is a number for us in the reals because it has a digit sequence. 0.10110111... is also a number for us now, and if we form the number line of reals, it is no longer gapy. Because this is the number line that we like, from now on we when we talk about "the number line" it is assumed that we mean this one with no gaps.

Question: As usual, how do we define operations on reals?

Answer: Effectively, we obviously cannot act on all of the digits of a number if they do some crazy, unpredictable pattern like for π. (By the way, I have not really formally introduced π and am only hinting that it is irrational--the proof is hard and we will save it for much later. For now I only ask you to have faith and patience with π.) More realistically we will just leave most irrational numbers in their symbolic form, like sqrt(2) or `\sqrt{2}`, rather than try to write down any part of their digit expansions. The operations will correspondingly just be left in symbolic form, like sqrt(2) + 1 most of the time. However this introduces the complication that some symbolic expressions are better than others. For instance `sqrt{2}\^^2` is better expressed as 2 since the square and square-root [are built to] cancel. We might call this simplification but the term will not be well-defined since in some cases it won't be clear what simplification means. Take `sqrt{8} = \sqrt{4\ast 2} = \sqrt{4}\ast \sqrt{2} = 2\sqrt{2}`. Which is "simpler", `sqrt{8}` which uses only one operation, or `2sqrt{2}` which uses smaller numbers in the operations of square-root and multiplication? Most high school classes teach that the later is simpler, but for me the question is useless. If you can make an expression obviously simpler, do it--but if there's any doubt about whether one way of writing a number down is any simpler than another way, it doesn't matter, move on to more interesting questions! Do Math not decoration!

That is the response about effective computations on symbols, although there may be a slightly different and deeper question of what even is--what sense can be given, in general, to say 21.75904^4.33156 or any of the other combinations of symbols in our new language? Well for finite decimals we are still in the case of rational number operations, and we can always decrypt these things into their corresponding rational expressions and perform any of sums, products, powers, and roots that are needed. For infinite decimals, for most usual cases we will have faith that if you just keep completing their digit sequences you will eventually hone in on some point in the number line. That is to say if we wanted to compute (0.10110111...)^2 we could first approximate this by 0.1^2 and then 0.101^2 and so on. At every stage you get a number and this sequence of numbers keeps honing in on a point in the number line which is the ultimate value of the original expression.

But to reiterate, we don't effectively compute things like this. Instead we typically just stick to symbolic representations and try our best to find the "simplest" representation available, whatever we might pick for that.

Question: Are the reals dense? That is to say, between any two reals r < s, must there be a t such that r < t < s?

Answer: Yes, if r and s are any two reals we can in fact construct a number in-between them. We can construct a lot of numbers, but the simplest is their average `\frac{r+s}{2}`. Intuitively this is equally distant from r and s. Here I'll prove that `r < \frac{r+s}{2}`. By multiplying by 2 and then subtracting r we get `r < s` and if we run this calculation in reverse we get an effective procedure for building the proof.

$$r < s \Rightarrow 2r < r+s \Rightarrow r < \frac{r+s}{2}$$

A similar proof shows that `\frac{r+s}{2} < s`.
