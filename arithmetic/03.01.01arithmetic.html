Chapter 03: Alternative systems

Section 01: Modular arithmetic

Abstract: If we cycle our counting, like counting hours on a clock, we get a new number system. The numbers in the system are congruence classes, which already have negative elements in a sense. Some elements also have reciprocals.

Terms: Equivalent mod n, congruence class, integers mod n, zero divisors, Euclidean division algorithm, minimum (of a set)

Symbols: ≡n, [.], ℤn, minA

Question: An ancient Chinese text contained the question "There are certain things whose number is unknown. If we count them by threes, we have two left over; by fives, we have three left over; and by sevens, two are left over. How many things are there?" Well?

Answer: This is a hard problem and we can run through all numbers until we eventually find that 23 satisfies this problem since 23 mod 3 = 2, 23 mod 5 = 3, 23 mod 7 = 2. It's not the only number that does, also 128 satisfies these relations too. In general all numbers 23+105n satisfy this question. We would like to find a more efficient algorithm for this than simply trying each number, but as usual, we need to break the problem down into simpler parts. An important tool is to define the notion of modular arithmetic.

For any given positive integer n we define the set of integers mod n. In a sense this is like restricting our arithmetic down to just the numbers 0, 1, ..., n-1. If we ever exceed these numbers we cycle back to 0. This is really just finding the remainders mod n. Take for instance n = 5, then we restrict to the numbers 0, 1, 2, 3, 4. If we ever compute 4+1 we say that this is congruent to 0 mod 5 and write 4+1 ≡5 0. Similarly 4+2 ≡5 1 and 3+3 ≡5 2 and so on, which defines the idea of two numbers being equivalent mod n. This means that 0 ≡5 5 ≡5 10 and so on. In the same vein we can say 1 ≡5 6 ≡5 11 and so on. Thus we define a notion of a congruence class. The congruence class of 0, written [0]5, is the set of all things congruent to 0. Hence [0]5 = {m∊ℤ : m ≡5 0}. Notice that this includes negatives like _5 since _5 mod 5 = 0. If that last sentence was not immediately obvious you may want to review the definition of the mod function given in the first chapter of this topic.

With this notation we can also say [1]5 = {..., _4, 1, 6, ...} and likewise for the three other congruence classes. We say that the set of all congruence classes, {[0]5, [1]5, [2]5, [3]5, [4]5}, is the integers mod 5 and we denote this by ℤ5. To be sure the point is clear, I'm saying ℤ5 = {[0]5, [1]5, [2]5, [3]5, [4]5}. Likewise we define the integers mod n as the set of equivalence classes [0]1, ..., [n-1]n.

Notice that this system has no real need for subtraction because there already exist numbers that sum to 0. 4+1 ≡5 0 so that in a sense [4]5 is the negative of [1]5. By a similar logic [3]5 is the negative of [2]5.

Exercise: List the congruence classes of ℤ3 and determine what 71 is congruent to mod 3.

Answer: ℤ3 = {[0]3, [1]3, [2]3]} and we can find which one 71 is in by shaving off this number by 3s or by dividing it by 3 and finding the remainder. The latter sounds easier so let's use the fact that 71 mod 3 = 2. Hence 71 ∊ [2]3.

Question: How can we add numbers in ℤn? For instance how can we easily compute [12]4 + [100]4 + [137]4? And what about multiplication?

Answer: It makes some sense that we could compute each mod and then sum the results, and then computing another mod if needed. This is easier than first summing 12+100+137 and then computing the mod because summing makes the number much larger and therefore harder to mod. If it is legitimate mod first then we could compute [0]4 + [0]4 + [3]4 and immediately obtain that the answer is [3]4.

So can we be assured that we can mod first and then sum rather than sum and then mod? Sure, here's a proof. If w ≡n x then there must exist integer m such that w = mn + x. This should be clear from the definitions in terms of remainders. Notice the switch from congruence to equality; now that we have equality, we are justified in using substitution of equals. Likewise if y ≡n z then there is some p such that y = np + z. We want to show that if you first sum w+y it'll be in the same if you first found w's congruence class (which is x) and y's (which is z) and summed the congruence classes. Thus we want to show w+y ≡n x+z. This is the same as saying that we want to show that there is some integer q such that w+y = nq(x+z). Now that's a clear statement of what we want. Getting back to what we have, w = mn + x and y = np + z, if we then sum w+y we get

$$w+y = mn+x+np+z = n(m+p) + x + z$$

But since m+p is an integer, it can serve as our q and this ends the proof.

For multiplication much the same story is true, we can first mod and then multiply if we want--and usually, we want. The proof that this is legitimate will require us to show wy ≡n xz which is the same as there existing a q such that wy = nq + xz. The proof is

$$wy = (mn+x)(pn+z) = mnpn+mnz+xpn+xz = n(mnp + mz+xp) + xz$$

In this we choose q = mnp + mz + xp. Thus if we want to compute [12]4 * [100]4 * [137]4 we get the same result as by computing [0]4 * [0]4 * [3]4 which is clearly [0]4.

Question: What about division?

Answer: It turns out that division in this system is quite different from how we construct it with normal numbers. Recall that p/q means the number which, when multiplied by q, produces p. I said this perspective would be important and by now this must be the fifth time at least that I've made important use of the idea.

Anyway, suppose that we work in ℤ3. I'm getting tired of writing all these subscripts so I'll just drop them. Context should determine which module I'm using, and if it's not then I'll put the subscripts back on. What is for instance [1]/[2] in ℤ3? It doesn't take checking a lot of cases before you find that [2][2] = [4] = [1]. Therefore [1]/[2] = [2]! Very strange this system where a number (other than 1 or _1) is its own reciprocal. We write the reciprocal of [2] with the superscript notation [2]^_1 = [2], hearkening back to our notation for reciprocals in other contexts. Unfortunately, for now, the only good way to find a number's reciprocal is to cycle through all of the possible products in the set.

Exercise: Find [5]^_1 and [7]^_1 if we work in ℤ9. Does every number in the set have a reciprocal?

Answer: [5][2] = [10] = [1] so [5]^_1 = [2]. By a similar calculation you can show [7]^_1 = [4]. Not all numbers have reciprocals. [0], [3], and [6] don't, and it's no accident that these are the numbers which share a common factor with 9. These are also the numbers which you can multiply by something to get [0]. So for instance [3][3] = [9] = [0] and [6][3] = [18] = [0]. For this reason we call [0], [3], and [6] zero divisors. We give the name units to the numbers that are not zero divisors so that, mod 9, the numbers [1], [2], [4], [5], [7], and [8] are units.

Exercise: Find the units and zero divisors of ℤ10.

Answer: The units are [1], [3], [7], and [9]. Everything else is a zero divisor.

Question: How can we characterize the units and/or zero divisors?

Answer: The units are the numbers coprime with n in ℤn! I told you the concept of coprime would return! Since 1, 2, 4, 5, 7, are all coprime with 9, then these will all be units in ℤ9 and everything else a zero divisor.

As a small note, to sort of exercise the concept, notice an easy consequence of this claim. If n is prime, then every number is a unit except [0].

In any case, this is a very important fact and we're about to study it somewhat deeply. The claim is that [m] is a unit mod n if they're coprime. This is equivalent to saying that, if they're coprime, there is another number, call it [a], such that [m][a] = [1]. This last equation is equivalent in terms of integers to saying there is an integer, call it b, such that am + bn = 1.

This important fact is a special case that has been generalized in Bezout's identity. This tells us that, even if m and n aren't coprime, we can still get am+bn = gcd(m,n). We will delay the proof of this theorem until the end of the section, but in the mean time we will exercise the use of the theorem to get clear on what it means.

Exercise: Find gcd(2,3) and find numbers a, b such that 2a + 3b = gcd(2,3).

Answer: In this case gcd(2,3) = 1 because they're coprime. Clearly one of a, b is going to have to be negative, and it's very easy to see that a=_1 and b=1 work. With this choice we get 2a + 3b = 1.

Although the numbers are small enough that the answer was easy to obtain, let's start using the so-called Euclidean division algorithm. To do this, we first divide the larger of the two numbers, 3, by the smaller, 2, getting the quotient and remainder. Hence we find

$$3 = 2(1) + 1$$

Because the numbers are so small, at this point, we already move the term with 2 in it, to the other side of the equation.

$$2(\_1)+3(1) = 1$$

Exercise: Find gcd(10, 25) and find a, b such that 10a + 15b = gcd(10,15). Use the division algorithm.

Answer: gcd(10,15) = 5. This time a = _2 and b = 1. The division algorithm goes

$$25 = 10(2) + 5 \Rightarrow 10(\_2)+25(1)=5$$

Exercise: Find gcd(234, 432) and find a, b such that 234a + 432b = gcd(234, 432). Use the division algorithm.

Answer: gcd(234, 432) = 18. This time the numbers are so hard to find, it will be easier to start with the division algorithm in order to find a, b.

$$432 = 234(1) + 198$$

But since we do not yet have the gcd in our expression, we need extra steps in the division algorithm. To do so, we make a new equation with only the divisor and remainder from what we found earlier.

$$234 = 198(1) + 36$$

Again we don't have the gcd so we do it again.

$$198 = 36(5) + 18$$

If we didn't already know that 18 were the gcd, we could confirm it by further computing 36 = 18(2)+0. Whenever we reach a line with no remainder, we have found the gcd.

Now that we have the gcd, we reach the end of this stage of the algorithm and move into the second stage. In this second stage, we work from the last equation to the first, solving for remainders and substituting. The last equation solved for its remainder is

$$198-36(5) = 18$$

We will then keep substituting numbers into the left-hand side until we get an equation with the gcd. We always substitute the remainders that we found at each equation, so to begin with, we substitute 36.

$$198 - (\overbrace{234-198}^{36})(5) = 18$$

and next sub the previous remainder, 198,

$$\overbrace{432-234(1)}^{198}-(234-(\overbrace{432-234(1)}^{198}))(5) = 18$$

This is an equation that has only our two numbers, 234 and 432, together with a bunch of factors. If we simplify, we'll get the a and b values that we're seeking.

$$432 - 234 - (234 - 432 + 234)(5) = 18\\ 432 - 234 - (234(2)-432)(5) = 18 \\ 432- 234(10) + 432(5) = 18 \\ 234(\_10) + 432(6) = 18$$

Hence a, b = _10, 18 respectively.

Question: What is the proof of Bezout's identity?

Answer: The Euclidean algorithm is almost the whole proof of Bezout's identity! I mean, it always terminates at the gcd, and by construction it necessarily solves for the gcd in terms of m and n. The only serious gap in the proof is the claim that the algorithm terminates at the gcd--if we could seal that hole, this would be a pretty good proof. Note that in this section my goal is to give the most intuitive proof that I can and therefore may skimp on some rigor. I'll present a more thorough proof in the Number Theory chapter.

So how can we know that the Euclidean algorithm always terminates at the gcd? To do so we have two parts to prove: i) That the number it terminates at divides both of the original numbers, and ii) that it is the greatest such number. Let's call the number d, which the algorithm terminates at. This is to say, we eventually reach a line that reads c = dq for some appropriate c and q. For part (i) we want to prove that d divides both m and n. In the line just above the one where c = dq appears, we will have another line like c2 = cq2 + d, i.e. one where d features as the remainder and c the divisor, as this is the nature of the algorithm. But in that case, d divides the right-hand side, i.e. d divides cq2 + d. __Hide__ Therefore d divides the left-hand side since it's equal, i.e. d|c2. Well at this stage we can go up a line, say to c3 = c2q3 + c and repeat the basic argument: c divides the right-hand side and therefore it divides the left-hand side. And so on, until we reach the top two lines, which contain our original numbers. Thus we have a chain of divisibility relations that have at their bottom d. I.e. d|c|c2|c3|...|m|n. Since divisibility is a transitive relation __Hide__ then d divides m and n. So we have proved part (i).

Next we need to show that d is the largest common divisor, that is to say if e also divides both numbers then e ≤ d. This way of mathematically stating "largest such that blah blah blah" is common: Show that it has the "blah blah blah" property and then show that anything else that has the "blah blah blah" property must be less-than-or-equal-to. So how do we do it? We have already shown that d = am + bn. If e|m and e|n then e must divide the right-hand side because it divides both summands. Therefore it divides d, the left-hand side. But if e|d then e ≤ d. __Hide__

Question: What does any of this have to do with the question that began our studies?

Answer: Let's again take a simpler version of the problem, like when a quantity is grouped by 3s there are 2 left over and when grouped by 4s there is 1 left over. This is a system of congruence equations.

$$x \equiv_3 2 \\ x \equiv_4 1$$

We can determine x up to a multiple of 12, but after 12 the congruence patterns repeat. To demonstrate this, look at how the pattern in the aligned numbers repeats below.

__Example__

This demonstration above shows all combinations of remainders mod 3 and 4. So 0 has remainder 0 for both. 1 has remainder 1 for both. Five columns down shows us that 4 has remainder 1 mod 3 and remainder 0 mod 4. As you can see, after 12, the pattern of remainders repeats and therefore we can only answer this question up to a multiple of 12. So be it, we will solve this up to mod 12.

One (of a great many) thing that Bezout's identity gives us is what I said before, that every m coprime to n has a reciprocal in ℤn, i.e. there is some [a] such that [a][m] = 1. Since 3 and 4 are coprime (in fact, prime) then we can find their reciprocals mod the other. 3^_1 ≡4 3 and 4^_1 ≡3 1. What this does for us is allows us to slip in a factor of 3*3^_1 = 3*3 = 9 in the second equation and 4*4^_1 = 4*1 = 4 in the first, because in each congruence relation this is like inserting a factor of 1.

$$x \equiv_3 2*4 = 8 \\ x\equiv_4 1*9 = 9$$

So ... what does this get us? Notice that 8 ≡4 0 and 9 ≡3 0. What this means is that we can add these to each congruence equation without changing it.

$$x \equiv_3 8 + 9 \\ x \equiv_4 9 + 8$$

From this it follows that `x \equiv_{12} 17 \equiv_{12} 5`. Lo and behold 5 solves the system of congruence equations.

Exercise: Let's try this with a larger collection of congruence equations. Solve

$$x \equiv_5 3\\ x\equiv_7 5 \\ x\equiv_{8} 0$$

Answer: Again we cannot solve this system except up to mode 5*7*8 = 280, after which the pattern of remainders repeats. Last time we found the reciprocal of one number mod the other and so on. This time things are not quite as clear.

We want to again insert a factor into the first equation that, mod 5, is congruent to 1; but mod 7 and 8 is congruent to 0. A natural number to find therefore is 56^_1 mod 5. This is easy since 56 ≡5 1 so we already have 56^_1 = 1. Thus we change the first congruence equation to x ≡5 3*56 = 168.

Similarly we find 40^_1 mod 7. This one's hard so we can use Bezout's identity to find it.

__Example__

Since 3*40 - 7*17 = 1 then 3*40 ≡7 1 so 40^_1 = 3 mod 7 and therefore we make the second congruence equation x ≡7 5*40*3 = 600.

For the final congruence we could perform similar labors but since it's congruent to 0, then multiplication by anything will stay 0. At this point we sum all of the numbers and set x ≡280 168+600 ≡280 208.

And amazingly if we check, it all works out. 208 mod 5 = 3 and 208 mod 7 = 5 and 208 mod 8 = 0.
