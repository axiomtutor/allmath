0. The guiding questions

In this section we discuss numbers, how to add, and so on, but also more complicated subjects like binary representation and eventually, some basics about complex numbers. To frame this in a concrete context, suppose you have a certain number of bags of grain in different warehouses and would like to tally them up to know the total: How could you efficiently do so? Also suppose that you need to distribute these to peasants in order to prevent a riot--do you have enough? Along the way, seeing the different ways we can choose to represent numbers will tell us something about a very modern subject, binary numbers.

Finally, what should we do if certain operations on numbers, like 7/3, do not have values in our standard number systems?

1. Counting with numbers

Busy idiot summary: Most number systems are in a base, which we use for counting by cycling when the base symbol set is exhausted, e.g. 0, 1, 10, 11, 100, ... in binary. Each digit then represents a count of a power of the base. This makes easy methods of tabular calculation for addition, subtraction, multiplication, and division. My canonical example of addition is combining baskets of apples; subtraction is inverse addition, and my canonical example of subtraction is eating; multiplication is repeated addition, and my canonical examples of multiplication are either many same-size baskets of apples, or arranging objects in grids; division is inverse multiplication, and my canonical example is grouping apples in baskets.

How are you going to answer this quickly? Delete strokes from the grain and peasant lists until you see which has strokes left over? (Assuming every sack of grain will satisfy one peasant.) This process is horribly inefficient and basically equivalent to the Roman numeral system. Roman numerals had a slight advantage in that they would group fives and tens together with a single symbol like V and X, and so on. Still, there's an obvious better way, one we all use every day. Hindu-Arabic numerals, the numerals that we use, 0 through 9 arranged in digit places, do a much better job handling large numbers. Using these we might write 60 and 33 to count sacks of grain, where the first number has a 6 in the tens-place to represent 6 groups of 10. Roman numerals don't use digits places and therefore they had to try adding numbers like VIXX with XXIX. If I had to do that I would probably prefer to convert both numbers to my numerals and then add.

There's a better way and, in fact, the Sumerians knew about it millenia before the Romans existed. The Sumerians were one of the few civilizations in the world to independently invent writing, and quickly developed a number system with a base. To help understand what this means, realize that our number system (called the Hindu-Arabic numerals) uses a base-10 system. We have a symbol for ten numbers, 0 through 9, and when any digit "fills up" we cycle it back to 0 and set the next digit up 1. The Sumerians (also called the Babylonians) used a base-60 system. That means they had a symbol for every number up to 59 and then cycle when the digit fills up. That's a little deceptive because the symbol they had for 1 was | and the symbol they had for 2 was ||, and so on. Basically up to 59, they had a Roman numeral-like system. But at 59, they would cycle like we do at 10. This is where we get 60-second minutes and 360 degrees in a circle.

Importantly this is very similar to how computers represent numbers in base-2 or binary numbers. Here they use only on-off, or 0 and 1. In fact, what this really is in a computer is high-low current, but we'll study that much later when we learn about Physics. I will use the special font style that you see above for binary numbers in order to distinguish them from decimal (i.e. Hindu-Arabic) numbers. In any case, throughout this and many other sections we'll use only base 10, base 2, and base 16 systems. Base 16, called hexadecimal, is also used in certain computer languages. Hopefully the pattern is clear. In base 10 a number in the "tens place" represents count of the groups of ten so that 16 means 1 group of 10 plus 6 single units. The number 432 represents 4 groups of one hundred, plus three groups of ten, plus 2 singles. This goes by powers of 10 and the same is true for binary numbers. The powers of ten are 1, 10, 100, 1000, and so on. The powers of two are 1, 2, 4, 8, 16, etc. In binary the number 101 represents 1 group of fours, no groups of two, and a single unit. Therefore if we want to represent it in Hindu-Arabic numerals it would be 5.

B. Computer and table methods of computing

I. Adding and subtracting

Now that we can count, how do we add? If we use Hindu-Arabic numerals to say that there are 567 sacks of grain in one store and 45 in another, how many sacks are there in total? We need to group the ones of 567 (i.e. we need 7) together with the ones of 45 (i.e. together with 5). Together these make 12 of course, but now that we have formed an extra group of tens we need to carry it over to the tens place. At this point we have to combine 6 and 4 and 1, all together making 11. But with more than ten groups of ten (i.e. more than 100) we can section off that 100 and carry it to the hundreds digit, which is now 6. All together that makes 612. All of these moves can be organized in a table method.
